# Daily Briefing Generator — Bee v4.0

Follow these steps to generate the daily briefing. Output goes to `context/daily/YYYY-MM-DD.md`.

## When to Run
- **Session start** — Every Claude Code session, as part of the Bee Operating Protocol
- **On demand** — When user says "briefing", "morning update", "what's today look like"
- **Skip if** — A briefing for today already exists at `context/daily/YYYY-MM-DD.md`

## Data Gathering (run these MCP calls in parallel where possible)

### 1. ClickUp — Overdue Tasks
```
clickup_search:
  keywords: ""
  filters:
    task_statuses: ["active", "unstarted"]
    due_date_to: [yesterday's date, YYYY-MM-DD format]
  count: 20
```
Flag anything overdue as URGENT.

### 2. ClickUp — Due Today
```
clickup_search:
  keywords: ""
  filters:
    task_statuses: ["active", "unstarted"]
    due_date_from: [today, YYYY-MM-DD]
    due_date_to: [today, YYYY-MM-DD]
  count: 20
```

### 3. Google Calendar — Today's Events
```
gcal_list_events:
  timeMin: [today]T00:00:00
  timeMax: [today]T23:59:59
  timeZone: Asia/Jerusalem
```

### 4. Gmail — Important Unread
```
gmail_search_messages:
  q: "is:unread is:important"
  maxResults: 5
```

### 5. Context File — Financial Alerts & Carry-Forward
Read `context/current-state.md` and extract:
- **Financial Alert** section (any deadlines, expired approvals)
- **Active Carry-Forward** section (all items with their escalation emoji)
- **Revenue Status** (current streak, target)

## Output Template

Write the following to `context/daily/YYYY-MM-DD.md`:

```markdown
# Daily Briefing — YYYY-MM-DD (Day of Week)

## Urgent
- [List overdue tasks, financial deadlines, and 🔴 carry-forward items]
- [If nothing urgent: "Nothing urgent today."]

## Today's Schedule
- HH:MM — Event name (location if any)
- [If no events: "Calendar is clear."]

## Due Today
- [ ] Task name — [list name] — [priority]
- [If nothing due: "No tasks due today."]

## Inbox (important unread)
- From: [sender] — [subject] ([relative time])
- [If inbox clear: "No important unread emails."]

## Carry-Forward
- [emoji] Item — status — due date
- [Show all items from current-state.md carry-forward section]

## Revenue Check
- Streak: X/5
- Next revenue action: [first pending revenue task from carry-forward]
- Revenue target: ₪7,000/mo | Current: ₪X/mo

---
*Generated by Bee v4.0 at HH:MM IST*
```

## Urgency Rules

An item is **urgent** (appears in the Urgent section) if ANY of these:
- Task is overdue (past due date)
- Task due today with priority URGENT or HIGH
- Carry-forward item has 🔴 escalation
- Financial alert has a deadline within 3 days
- Calendar event within 2 hours

An item is **skippable** (omit from briefing entirely) if:
- Task is in `closed` or `done` status
- Email is read
- Event is all-day and informational (holidays, birthdays)

## After Generation

1. Present the briefing content to the user directly in the chat (don't just say "file created")
2. Save to `context/daily/YYYY-MM-DD.md`
3. If there are urgent items, call them out explicitly: "You have X urgent items today"
4. Ask: "Does this session move us toward ₪7K/mo?"
